Data Overview

After gaining access, it is essential to develop a thorough understanding of the EHR data to assemble the elements of interest and create a dataset aligned with the research objectives. This typically involves answering the following questions:

  1. What data is available, and where is it stored?
  2. Which data elements are relevant, and how are they defined?
  3. How is the data organized and linked across files and tables?

These questions are typically addressed by consulting institution-specific EHR documentation or the warehouse data manager. For MIMIC-IV, comprehensive documentation is available here.

Tip

For users new to EHR data, we highly recommend watching the MIMIC data tutorial here.

What Data is Available?

Electronic Health Record (EHR) data can take many forms. For example, billing information and vital signs are typically recorded as tabular data, clinical notes are in free text format, ECG recordings generate waveform data, and procedures such as X-rays generate image data.

MIMIC-IV data comes in “modules” based on the source of data generation:

  • hosp — Hospital-level data for patients, including labs, microbiology, and electronic medication administration records
  • icu — ICU-level data, including event tables identical in structure to MIMIC-III
  • ed — Data from the emergency department
  • cxr — Lookup tables and metadata from MIMIC-CXR
  • note — De-identified free-text clinical notes

Linking Data Across Tables

In EHR data, information is spread across multiple tables. Three major identifiers are used in MIMIC:

  • SUBJECT_ID — Patient-level identifier. The patients table contains demographics for each unique patient.
  • HADM_ID — Hospital admission-level identifier. Each unique hospital admission gets a unique ID.
  • STAY_ID — ICU stay identifier. All ICU admissions within 24 hours of each other are grouped.

Diagnoses Data

In U.S. healthcare systems, diagnoses are documented using ICD (International Classification of Diseases) codes. Two versions are in use:

  • ICD-9: used until October 1, 2015
  • ICD-10: starting October 1, 2015

The same diagnosis uses different codes depending on when it was recorded. For example, essential hypertension is 4019 under ICD-9 and I10 under ICD-10.

Batch Summarization Function

For larger datasets (e.g., lab data with 158M+ rows), we define a batch summarization function:

def get_basic_summary(filepath, cols_to_summarize, batch_size):
    frequencies = defaultdict(lambda: defaultdict(int))
    non_null_records_count = defaultdict(int)
    total_records_count = 0

    for batch in pd.read_csv(filepath, usecols=cols_to_summarize,
                             chunksize=batch_size, dtype=str):
        total_records_count += len(batch)
        for column in batch.columns:
            non_null_records_count[column] += batch[column].notnull().sum()
            value_counts = batch[column].value_counts()
            for value, count in value_counts.items():
                frequencies[column][value] += count

    freq_table = [
        (column, len(frequencies[column]), non_null_records_count[column])
        for column in frequencies
    ]
    summary = pd.DataFrame(freq_table,
                           columns=["Column_Name", "Unique_Values", "Non_Null_Records"])
    return {'summary': summary, 'column_freq': frequencies, 'records': total_records_count}

Reusable Summary Functions

We define two reusable functions for generating comprehensive summaries across all data types:

def generate_wordcloud(sorted_freq_df, code_desc_column):
    word_freq = {}
    for index, row in sorted_freq_df.iterrows():
        title = row[code_desc_column]
        count = row['counts']
        word_freq[title] = word_freq.get(title, 0) + count

    wordcloud = WordCloud(
        width=1200, height=800, background_color='white',
        colormap='viridis', prefer_horizontal=0.9, max_words=100,
        contour_width=1, contour_color='black'
    ).generate_from_frequencies(word_freq)

    plt.figure(figsize=(12, 8))
    plt.imshow(wordcloud, interpolation='bilinear')
    plt.axis('off')
    plt.tight_layout(pad=0)
    plt.show()
    return wordcloud


def file_line_count(filepath):
    count = 0
    with open(filepath, 'r') as file:
        for line in file:
            count += 1
    return count
